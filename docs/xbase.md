# xBase

> 项目探测 + skill 初始化 + 状态管理

**核心文件**

- **SKILL-STATE.md**（状态枢纽）
  所有 skill 的初始化状态、核心文件路径、探测结果。系统预设文件，非运行时产物。
  - **结构**：`## 项目信息`（共享路径）+ 每个 skill 一个 `## xName` 段
  - **字段**：核心文件路径 + `initialized` 日期
  - **写入**：仅 xBase
  - **读取**：所有 skill 启动时
  - **更新时机**：`init` / `reinit <skill>` / `ensure` 补充未初始化的 skill
- **run.sh**（工具）
  项目的构建/启动/停止/日志读取脚本，封装项目特定命令。
  - **创建**：xBase 初始化时，根据探测到的构建系统生成
  - **读取**：xDebug（构建/启动/日志读取）、xTest（构建/启动）
  - **更新时机**：项目构建系统变更时，通过 `reinit xbase` 重新生成

### D1. 运行模式
用户指定或默认的初始化范围。
- **status**（用户请求查看状态）→ 展示每个 skill 的初始化状态、核心文件路径（终点）
- **ensure**（默认）→ 只处理未初始化的 skill → D2
- **reinit \<skill\>**（指定 skill 需重新初始化）→ 重新初始化该 skill → D2
- **init**（全量重置）→ 重置所有 skill → D2

### D2. 探测与确认
通过标志文件（如 package.json / Cargo.toml / *.xcodeproj）自动识别语言、框架、构建系统、日志框架、测试框架。探测结束后展示每个核心文件的四态判定结果，等待用户确认后进入创建阶段。遵循系统设计原则中的"初始化"三条（探测优先、幂等安全、增量补全）。
→ D3

### D3. 核心文件处理
逐文件判断状态并处理。
- **需创建**（文件不存在）→ 按格式定义生成
- **迁移候选**（文件存在但结构不匹配当前格式定义）→ 提示用户选择迁移或保留
- **需更新**（文件存在且结构匹配，但内容过时）→ 合并更新，保留用户自定义内容
- **已就绪**（文件存在且内容与探测结果一致）→ 跳过

全部文件处理完成后进入清理阶段：处理废弃文件（迁移候选的旧文件）+ CLAUDE.md 中与核心文件重复的规范段落。汇总结果。

# xDecide

> 决策守护：轻重分流 + 前提审计 + 偏见检查 + 决策演化追踪

**核心文件**

- **DECIDE-LOG.md**（记录）
  所有技术/产品决策的结构化记录——背景、前提假设、方案对比、结论和理由。支持前提审计（假设是否仍成立）和决策演化追踪（Supersede 链）。
  - **创建**：xBase→xDecide 阶段 0，按 template 创建空文件
  - **写入**：xDecide 新决策/替代决策时按 standard 追加条目
  - **读取**：xDecide 回顾审计时；xDebug 修复时查询相关决策
  - **更新时机**：每次新决策 / Supersede 旧决策 / 前提审计标记过时

**参考文件**

- **decide-log-template.md**（模板）
  DECIDE-LOG.md 的结构骨架——文件头和条目占位符。
  - **消费场景**：xBase→xDecide 阶段 0 创建 DECIDE-LOG.md 时
- **decide-log-standard.md**（书写标准）
  DECIDE-LOG.md 的书写标准——条目字段、编号规则、修订格式、三态检测。
  - **消费场景**：xDecide 写入新决策/替代决策时

### D1. 操作类型
- **新决策** → D2
- **回顾与审计** → D4
- **替代（Supersede）** → D5

### D2. 决策级别判断
AI 基于影响范围和可逆性判断。
- **轻量决策**（日常小决策）→ Y-Statement 一句话录入（30 秒完成）（终点）
- **完整决策**（重大/不可逆）→ D3

### D3. 完整决策流程
引导式流程：
- **理清背景与约束**：当前状态、技术约束、业务需求
- **显式化前提假设**：明确列出"此决策基于以下假设：..."，让未来可审计
- **穷举方案+利弊分析**：扫描代码+搜索历史决策，至少列出 2-3 个选项（含"不做任何改变"）
- **认知偏见检查**：主动挑战——是否考虑了不做任何改变？选错了后果可逆吗？可能忽略了什么？
- **做出选择+说明理由**：给出推荐，但明确标注"决策权在你"
- **预判后果**：正面和负面影响
- **代码关联建议**：扫描涉及此决策的代码，建议在决策实现点添加 `// See D-NNN` 注释

### D4. 回顾与审计
查看完整上下文（背景、前提、方案、结论）。检查前提假设是否仍然成立，标记疑似过时的决策。搜索代码中引用此决策的位置，评估决策的实际影响范围。回顾后需补充信息时在原条目追加修订段，不修改原始内容，保留完整演化历史。

### D5. 替代（Supersede）
新建决策，明确引用被替代的旧决策，解释环境变化。将旧决策状态设为 Superseded by D-NNN。保留完整的决策演化历史可追溯。

# xDoc

> 文档维护：巡检 + 同步更新 + 结构重组 + 内容瘦身

**核心文件**

- **DOC-RULES.md**（规范）
  项目的文档结构约定——目录组织规则、命名规范、代码-文档映射关系。
  - **创建**：xBase→xDoc 阶段 0，agent 在 guideline 指导下，从项目目录结构和 CLAUDE.md 提取生成
  - **读取**：xDoc 巡检和同步时以此为基准判断合规性和漂移
  - **更新时机**：项目文档结构变更时，通过 `reinit xdoc` 重新生成

**参考文件**

- **doc-rules-guideline.md**（生成指南）
  DOC-RULES.md 的生成指南——结构模板 + 质量原则，指导 agent 从项目数据产出高质量文档规范。
  - **消费场景**：xBase→xDoc 阶段 0 生成 DOC-RULES.md 时

### D1. 操作模式
- **巡检** → D2
- **同步** → D3
- **重组** → D4
- **瘦身** → D5

### D2. 巡检
优先运行 DOC-RULES.md 中配置的项目检查脚本，脚本覆盖不到的再用内置逻辑。检查五个维度：断链检测（内部/外部链接）、格式合规（标题层级、代码块标注、空文件）、结构合规（目录结构是否符合约定）、新鲜度评估（代码变更后未同步的文档）、覆盖率分析（无对应文档的代码模块）。输出健康报告（终点）。

### D3. 同步
分析最近代码变更（git diff / git log），基于代码-文档映射定位需要更新的文档（漂移定位）。直接草拟文档更新内容（不只是提醒），逐项确认 + 写入。
→ D6

### D4. 重组
对比当前结构 vs 约定结构，提出移动/合并/拆分建议。确认后执行重组：移动文件 + 自动更新所有引用 + 重建索引。
→ D6

### D5. 瘦身
冗余检测（多处描述同一概念但内容不一致）+ 过时检测（无用文档、已废弃内容）。提出删除/合并建议，逐项确认后执行。
→ D6

### D6. 批量修复验证
修改超过 10 个文件时，运行 DOC-RULES.md 中指定的验证脚本确认修复质量。

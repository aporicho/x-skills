# TEST-CHECKLIST.md 规范

## 创建位置

放在项目文档目录下（根据项目结构判断位置，如 `docs/`、`document/`、项目根目录等）。

## 文件格式

```markdown
# TEST CHECKLIST

> 由 /xtest skill 自动维护，勿手动编辑

更新时间：YYYY-MM-DD

## 概览

| 类别 | 总计 | ✅ 通过 | ❌ 失败 | ⏳ 待测 |
|------|------|--------|--------|--------|
| 🤖 自动化 | 0 | 0 | 0 | 0 |
| 👤 手动 | 0 | 0 | 0 | 0 |
| 🤝 结合 | 0 | 0 | 0 | 0 |

## 🤖 自动化测试

| ID | 测试项 | 状态 | 日期 |
|----|-------|------|------|

## 👤 手动测试

| ID | 测试项 | 状态 | 日期 | 备注 |
|----|-------|------|------|------|

## 🤝 结合测试

| ID | 测试项 | 状态 | 日期 | 备注 |
|----|-------|------|------|------|
```

## 编号规则

从项目代码结构动态推导 ID 前缀和编号体系，不硬编码。

## 迁移旧格式

如果检测到已有测试清单但格式不同：
1. 用 AskUserQuestion 展示差异，询问是否迁移
2. 用户同意 → 保留所有原始测试结果（通过/失败/待测状态），套用新格式
3. 用户拒绝 → 保持原样，后续更新时仍遵循原格式
